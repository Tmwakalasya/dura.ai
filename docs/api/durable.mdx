---
title: "@durable"
description: The decorator that turns a plain Python function into a durable, crash-proof workflow.
---

```python
from dura import durable
```

---

## Signature

```python
# Bare decorator — uses dura.db in the current directory
@durable
def workflow(ctx, *args, **kwargs): ...

# With options — set a custom log path for every call
@durable(log="path/to/run.db")
def workflow(ctx, *args, **kwargs): ...
```

`@durable` wraps a function into a durable workflow. The decorated function
receives a [`Context`](/api/context) as its first argument (`ctx`), injected by
the decorator — do not pass it yourself when calling the function.

---

## Decorator parameters

<ParamField path="log" type="str" default="dura.db">
  Path to the SQLite write-ahead log. All step results for all runs are
  committed here.

  ```python
  @durable(log="/var/dura/workflows.db")
  def book_trip(ctx, dest, amount): ...
  ```

  Can be overridden per-call with the `log=` call-time kwarg.
</ParamField>

<ParamField path="emit" type="str" default="None">
  dura Cloud ingest URL. When set, every run and step event (started, executed,
  skipped, failed, rolled back, completed) is streamed to this endpoint from a
  background thread.

  ```python
  @durable(emit="https://your-dashboard.example.com/api/ingest")
  def book_trip(ctx, dest, amount): ...
  ```

  **Emission is best-effort by design.** The local WAL stays the source of
  truth; events are queued to a daemon thread and posted fire-and-forget. A
  dead network, slow endpoint, or full queue can never slow down or fail the
  workflow — visibility errors are silently dropped.

  Can also be set with zero code changes via the `DURA_EMIT_URL` environment
  variable. If `DURA_EMIT_TOKEN` is set, it is sent as a
  `Authorization: Bearer …` header.
</ParamField>

## Call-time keyword arguments

These reserved kwargs are intercepted by the decorator and never passed through
to your function:

<ParamField path="log" type="str">
  Overrides the decorator-level log path for this single invocation. Use to
  isolate independent runs — one file per order, one file per user, etc.

  ```python
  book_trip("Tokyo", 412, log="orders/order_42.db")
  book_trip("Paris", 199, log="orders/order_43.db")
  ```
</ParamField>

<ParamField path="run_id" type="str">
  Overrides the auto-derived idempotency key for this invocation. Use when
  the same logical run may be triggered with slightly different arguments (a
  retry wrapper that appends a timestamp, a queue system that adds metadata).

  ```python
  book_trip("Tokyo", 412, run_id="order_42")
  # crash …
  book_trip("Tokyo", 412, run_id="order_42")   # resumes same run
  ```

  Without `run_id=`, the key is `sha256(f"{fn.__name__}:{json(args)}:{json(kwargs)}")[:12]`.
</ParamField>

<ParamField path="emit" type="str">
  Overrides the decorator-level ingest URL for this single invocation.
  Resolution order: call-time kwarg → decorator kwarg → `DURA_EMIT_URL` env var.

  ```python
  book_trip("Tokyo", 412, emit="https://your-dashboard.example.com/api/ingest")
  ```
</ParamField>

<ParamField path="crash_after" type="str">
  Testing hook: raise `SimulatedCrash` immediately after the named step commits.
  Used to simulate a crash at a known point in tests and demos.

  ```python
  book_trip("Tokyo", 412, crash_after="charge_card")
  # SimulatedCrash raised after charge_card commits.
  # The charge is in the log. Rerun without crash_after= to resume.
  ```
</ParamField>

## Run IDs in detail

dura derives a deterministic run ID from the function name and its positional
and keyword arguments:

```python
run_id = sha256(f"{fn.__name__}:{json.dumps(args)}:{json.dumps(kwargs)}")[:12]
```

Two implications:

1. **Same arguments → same run.** Crash-and-restart with identical arguments
   automatically resumes from the last checkpoint. No explicit IDs needed.

2. **Different arguments → different run.** Even a single changed argument
   produces a new run ID and starts a fresh workflow.

```python
book_trip("Tokyo", 412)    # run_id: "a1b2c3d4e5f6"
book_trip("Tokyo", 412)    # resumes run "a1b2c3d4e5f6"
book_trip("Tokyo", 999)    # new run — different amount
book_trip("Nairobi", 412)  # new run — different dest
```

## Attributes on the decorated function

<ResponseField name="workflow.last_context" type="Context">
  The [`Context`](/api/context) from the most recent invocation. Useful in
  tests to assert on `ctx.events` and `ctx.rolled_back` without threading
  context through your code.

  ```python
  book_trip("Tokyo", 412)
  ctx = book_trip.last_context

  assert ("charge_card", "executed") in ctx.events
  assert ("book_flight", "skipped") not in ctx.events   # wasn't replayed
  ```
</ResponseField>

## Exceptions

<ResponseField name="SimulatedCrash">
  Raised when `crash_after=` matches a committed step. Used exclusively in
  tests and demos — not raised in production.
</ResponseField>

<ResponseField name="StepFailure">
  Exception type you can raise from inside a step's `fn` to explicitly signal
  failure and trigger saga rollback. Using a bare `raise RuntimeError(...)` has
  the same effect — any exception from `fn` triggers rollback. `StepFailure` is
  provided as a typed alternative for callers who want to `except StepFailure`
  and distinguish workflow failures from other errors.

  ```python
  from dura import StepFailure

  ctx.step("validate", lambda: (_ for _ in ()).throw(
      StepFailure("card declined")
  ))
  ```

  Note: the original exception is always re-raised as-is — it is not wrapped in
  `StepFailure` or any other container.
</ResponseField>
