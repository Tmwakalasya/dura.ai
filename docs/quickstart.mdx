## Install

```bash
pip install dura
```

Zero external dependencies. Python 3.9+. The only storage is SQLite, which
ships with Python itself.

## The problem in one line

```python
charge_card()   # ← Stripe committed
# process dies
book_flight()   # ← never runs. charge never rolls back.
```

The customer is billed. The flight is never booked. You find out from support.

---

## The fix: two lines of code

```python
from dura import durable

@durable
def book_trip(ctx, dest, amount):
    charge = ctx.step("charge_card", lambda: charge_card(amount))
    flight  = ctx.step("book_flight", lambda: book_flight(dest))
    ctx.step("update_crm",   lambda: update_crm(flight))
    return flight
```

That's it. Every `ctx.step` is automatically:

- **Exactly-once** — the result is committed to a write-ahead log before the next line runs. A crash and restart skips completed steps.
- **Resumable** — calling the function again with the same arguments picks up where it died.
- **Rollback-able** — add `undo=` to any step; if a later step fails, dura unwinds completed steps in reverse order.

---

## See it survive a crash

### Step 1 — run until the crash point

```python
book_trip("Tokyo", 412, crash_after="charge_card")
# SimulatedCrash raised after the charge commits.
# The card has been charged. The log has it recorded.
```

### Step 2 — restart with the same arguments

```python
book_trip("Tokyo", 412)
```

**Output:**

```
[charge_card]  ↺ skipped — committed in prior run
[book_flight]  ✓ executed
[update_crm]   ✓ executed
```

The card is charged exactly once. The flight is booked. The customer is not double-billed.

## Add auto-rollback

If a step can fail and you want to undo what came before it, pass `undo=`:

```python
@durable
def book_trip(ctx, dest, amount):
    charge = ctx.step(
        "charge_card",
        lambda: charge_card(amount),
        undo=lambda: refund_card(charge["id"]),  # fires if a later step fails
    )
    # If book_flight raises — no seats, rate-limit, anything —
    # dura automatically calls refund_card(). The customer is never out of pocket.
    flight = ctx.step("book_flight", lambda: book_flight(dest))
    ctx.step("update_crm", lambda: update_crm(flight))
    return flight
```

## Check what's in the log

After any run you can inspect the SQLite log directly:

```bash
sqlite3 dura.db "SELECT step_name, status, result_json FROM steps;"
```

```
charge_card|done|{"id": "ch_1", "amount": 412}
book_flight|done|{"id": "fl_1", "dest": "Tokyo"}
update_crm |done|{"ok": true}
```

Each row is one committed step. A crash between rows means the next run starts
from the last committed row.
